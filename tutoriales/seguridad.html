Seguridad en Linux

1. Principios de seguridad en Linux

  • Mínimos privilegios: Cada usuario debe tener solo los permisos necesarios para su tarea.
  • Actualización constante: Mantener el sistema y paquetes actualizados reduce vulnerabilidades.
  • Seguridad por capas: Se recomienda combinar múltiples medidas de seguridad.

2. Gestión de usuarios y permisos

El control de accesos es esencial para evitar modificaciones no autorizadas.

  • Para listar usuarios en el sistema:
    cat /etc/passwd

  • Para modificar permisos de archivos:
    chmod 700 archivo
    • chown usuario:grupo archivo

3. Firewall y reglas de red

Un firewall ayuda a filtrar tráfico y proteger el sistema de accesos externos.

  • Uso de iptables para reglas básicas:
    sudo iptables -A INPUT -p tcp --dport 22 -j DROP

  • Uso de ufw(más sencillo):
    sudo ufw enable #Activar firewall
    sudo ufw allow 80/tcp #Permitir HTTP

4. Protección contra malware y análisis de seguridad

Aunque Linux tiene menor incidencia de virus, es recomendable usar herramientas de análisis:

  • ClamAV: Antivirus de código abierto. sudo apt install clamav
    clamscan -r /home

  • Chkrootkit: Detecta rootkits en el sistema. sudo apt install chkrootkit
    sudo chkrootkit

5. Cifrado de datos

El cifrado protege la información sensible. Algunas opciones incluyen:

  • EncFS: Para cifrar carpetas individuales.
    encfs ~/cifrado ~/acceso

  • LUKS: Cifrado de discos completos.
    sudo cryptsetup luksFormat /dev/sdX

6. Auditoría y monitoreo del sistema

Herramientas como auditd permiten registrar eventos de seguridad.

  • Para instalar y activar:
    sudo apt install auditd
    sudo systemctl enable auditd
