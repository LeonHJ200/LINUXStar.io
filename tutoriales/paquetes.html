Administración de paquetes en Linux

La administración de paquetes es un aspecto fundamental en Linux, ya que permite instalar, actualizar y eliminar software de manera eficiente. Cada distribución utiliza su propio gestor de paquetes, optimizado para su ecosistema.

1. ¿Qué es un paquete?

Un paquete es un conjunto de archivos necesarios para la instalación de un programa. Puede incluir ejecutables, bibliotecas, configuraciones y documentación.

2. Gestores de paquetes según la distribución

Cada distribución utiliza su propio sistema de gestión de paquetes:

  • Debian y Ubuntu: APT (Advanced Package Tool), trabaja con paquetes .deb.
  • Fedora y RHEL: DNF (Dandified Yum), administra paquetes .rpm.
  • Arch Linux: Pacman, maneja paquetes .pkg.tar.zst.
  • openSUSE: Zypper, especializado en paquetes RPM.
  • Gentoo: Portage, basado en compilación desde código fuente.

3. Comandos básicos de administración de paquetes

Debian/Ubuntu (APT)

sudo apt update
sudo apt upgrade
sudo apt install <paquete>
sudo apt remove <paquete>
sudo apt autoremove

Fedora/RHEL (DNF)

sudo dnf check-update
sudo dnf upgrade
sudo dnf install <paquete>
sudo dnf remove <paquete>

Arch Linux (Pacman)

sudo pacman -Syu
sudo pacman -S <paquete>
sudo pacman -R <paquete>
sudo pacman -Qe

openSUSE (Zypper)

sudo zypper refresh
sudo zypper update
sudo zypper install <paquete>
sudo zypper remove <paquete>

4. Instalación de paquetes desde código fuente

Algunas distribuciones permiten instalar software compilándolo desde el código fuente:

./configure
make
sudo make install

Este método es común en Gentoo, Slackware y distribuciones minimalistas.

5. Gestión de paquetes Snap y Flatpak

Para instalar software universalmente sin depender de la distribución, se pueden usar:

Snap:

sudo snap install <paquete>

Flatpak:

flatpak install flathub <paquete>

Estos sistemas permiten ejecutar aplicaciones en entornos aislados.
